GLOSSARY
Understanding your stats

Session

A coding session represents one interaction with Claude Code, from when you start until you close or switch contexts. Includes all messages and tool calls.

Message

Each message is an exchange between you and Claude - your prompts, questions, or commands and Claude's responses.

Tokens

Tokens are the basic unit of text processing. Roughly, 1 token ≈ 4 characters. Used for billing and model context limits.

Input Tokens

Tokens in your messages, prompts, and files you share. These are processed by Claude to understand your request.

Output Tokens

Tokens in Claude's responses. Generally cost more than input tokens.

Cache Read

When Claude reuses previously processed context from cache. Costs ~90% less than processing new tokens.

Cache Write

When new context is stored in cache for reuse. Slightly more expensive upfront but saves money on repeated exchanges.

Cache Hit Rate

Percentage of cache tokens that are read from cache vs. newly computed. Higher = more cost efficient.

Tool Calls

Actions Claude performs on your behalf - reading files, running commands, searching the web, etc.

Streak

Consecutive days where you had at least one session. Shows consistency in your coding workflow.
